# xfstests SCRATCH baseline: generic/* tests that require a SCRATCH device and
# pass against the clio FUSE filesystem. CI runs ONLY these via
# scripts/xfstests/run_ci_scratch_xfstests.sh and every one must pass.
#
# These are complementary to ci_baseline_pass.txt: those tests use only
# TEST_DIR, while these need a second, independently-formattable scratch fs.
# The clio FUSE fs can't be reformatted (no mkfs), and a single runtime serves
# one global namespace, so the scratch driver instead brings up TWO isolated
# persistent runtimes (one backs TEST_DIR, one SCRATCH_MNT) and wires xfstests'
# native fuse mount-helper support so ./check mounts/unmounts each device
# itself. No clio adapter changes are required for the harness.
#
# How this list was produced (not hand-curated): all 527 scratch-requiring
# generic/* tests were swept against the clio FUSE fs on the merged build
# (fallocate + mtime/atime + O_TRUNC fixes; runtimes given a 2 GB DRAM tier).
# 482 are notrun (fuse feature gaps: shutdown, dm targets, symlinks-on-fuse,
# ...), 9 FAIL and 11 HANG. The 25 below passed a discovery run AND two
# independent verification runs.
#
# Failing/hanging scratch tests (tracked, not run in CI):
#   FAIL: 003 120 344 346 403 452 460 519 615
#   HANG: 339 345 347 464 707
# Established 2026-07-03.
#
# 2026-07-04 update: the O_APPEND hang class 069/129/476/642 now PASS and are
# gated below. Root cause was an O(N^2) append path: each tiny O_APPEND
# allocated a new BlobBlock + burned a 4 KB bdev slab, so a multi-million-int
# append grew the block list O(N) (O(N^2) read/write scans) and hit ENOSPC.
# Fixed by BlobBlock capacity_/spare-fill (reuse a block's spare slab) +
# CLIO_IPC_MODE=SHM in the driver (cross-process 3k->27k ops/s). Verified twice.
# 551 + 750 remain NOT gated: they no longer hang but CRASH the client mount
# daemon ("Transport endpoint is not connected") under their much heavier stress
# (551 = aio-dio-write-verify, 750 = fsstress soak) -- a cross-process SHM
# transport robustness issue under load, NOT the append fix (embedded fsstress
# is clean) and NOT the O_APPEND pattern; tracked separately.
